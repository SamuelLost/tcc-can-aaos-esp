A function to initialize the spi bus This function initializes the spi bus and the spi device using the driver/spi_master.h from ESP32.

Parameters
misopin number for miso.
mosipin number for mosi.
sclkpin number for sclk.
cspin number for cs.
hosthost number for spi bus.
Returns
esp_err_t: Structure with the error code.
ESP_OK if the SPI bus was initialized correctly
ESP_FAIL if the SPI bus was not initialized correctly

This function checks the pins for the spi bus and the spi device.
If host is 1 the function will use the HSPI bus, if host is 2 the function will use the VSPI bus.
If the pins are not defined (-1) the function will use the default pins for the spi bus.

Function to transfer data through the spi bus This function transfers data through the spi bus using the driver/spi_master.h from ESP32. This function is used to write or read data.

Parameters
registerAddressaddress of the register to be written or read.
datavalue to be written in the register.
commandcommand to be sent to the device. Instructions are defined in datasheet.
Returns
esp_err_t: Structure with the error code.
ESP_OK if the data was transferred correctly
ESP_FAIL if the data was not transferred correctly

First, clear the transaction structure to start a new transaction. Then the configuration is done.
The flags allow the use of the rx_data[4] and tx_data[4] array to read and send data.
The cmd field is the instruction to be executed.
The default size is 8 bits, this depends on the device.
The addr field is the address of the target register.
The tx_data[0] field receives the byte to be written to the register.
